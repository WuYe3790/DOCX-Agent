## 对工具设计的直接建议

1. 表格工具需要支持嵌套表格定位。

   当前文档里封面信息表是嵌套在外层封面表格里的。全文 `.//w:tbl` 会同时返回外层表格和内层表格，所以 agent 工具返回定位时要明确：

   - 第几个表格。
   - 是否在另一个表格内部。
   - 行列坐标。
   - 单元格完整文本。

2. 插入行工具应复制相邻行结构。

   本次插入行保持了行高、列宽、边框和单元格数量。最小可行实现可以：

   - 找到目标 `<w:tr>`。
   - 深拷贝目标行或下一行。
   - 清空各单元格文本。
   - 写入新文本。
   - 插入到目标行之后。

3. 清空单元格工具要保留空段落。

   Word/WPS 清空单元格后保留了空 `<w:p>`，并删除文本 run。工具也应该这样做，避免生成没有段落的非法或不稳定单元格。

4. 删除行工具应删除完整 `<w:tr>`。

   删除整行和清空行内容不同。用户说“删除整行”时，工具应移除 `<w:tr>`；用户说“清空这一行”时，才保留行结构并清空单元格内容。

5. 表格内文本替换可以复用逻辑段落替换。

   `[TABLE-004-REPLACE-CELL]` 的变化和 002 的表格单元格替换一致：跨 run 命中后合并为一个 run，并继承原格式。现有 `replace_text` 逻辑可以保留，但建议后续增加表格坐标参数，让 agent 不必只靠全文搜索。

6. 行列格式最好和“列语义”绑定。

   封面信息表左列是字段名，右列是填写值。插入行时左列继承加粗，右列不加粗。后续工具可以提供：

   - `copy_format_from="above" | "below" | "same_column"`
   - `cell_texts=["...", "..."]`
   - `clear_existing_cell_runs=true`

   这样比让模型手动调用多个文本工具更稳定。
